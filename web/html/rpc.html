Testing it out from the terminal

Having installed SWI-Prolog 7 and added the executable to the path, we can start up SWI-Prolog with:

swipl

We load the pengines module, and call the predicate pengine_rpc/2, with the server-location and predicate-to-execute as the parameters.

?- use_module(library(pengines)).
?- pengine_rpc('http://localhost:3020',rdf(X,Y,Z)).

Prolog uses pattern matching to find the first occurence of an rdf triple with subject, predicate and object mapped to X, Y, and Z respectively. For this query the server replies with:

X = 'http://purl.org/dc/terms/hasPart',
Y = 'http://www.w3.org/2000/01/rdf-schema#label',
Z = literal(lang('en-US', 'Has Part'))

Pressing semi-colon (Prolog operator "or") gives the next matching result. Pressing period ends the RPC.

From your Prolog program

As in the "Testing it out from the terminal" example your module needs to load the pengines library module, and when you want to call the server you use the predicate pengine_rpc/2. As the developer you can add predicates that combine your application logic with the rdf/3 RPC call. You can then transform the resulting data into a suitable format for your application.
Here's a simple query for the Object "Acidose", mapping the resulting subject and predicate to variables S and P:

pengine_rpc('http://localhost:3020',rdf(S,P,literal('Acidose'))).

Result:

S = 'http://research.idi.ntnu.no/hilab/ehr/ontologies/icd10no.owl#E872',
P = 'http://www.w3.org/2000/01/rdf-schema#label'.
